# Markdown and MDX Guide

Learn how to write effective documentation for the gx402 SDK using Markdown and Mintlify's MDX components.

## Basic Markdown

### Headers
Use headers to organize your content hierarchically:

```md
# Main Topic (h1 - only one per page)
## Section (h2)
### Subsection (h3)
#### Minor Section (h4)
```

### Text Formatting
```md
**Bold text** for emphasis
*Italic text* for subtle emphasis
~~Strikethrough~~ for deprecated features
`Inline code` for code snippets
```

### Lists
```md
Unordered list:
- Item 1
- Item 2
  - Nested item

Ordered list:
1. First step
2. Second step
   1. Sub-step
   2. Another sub-step
```

### Links and Images
```md
[Link text](/relative/path) for internal links
[External link](https://example.com)
![Alt text](/path/to/image.png "Optional title")
```

## MDX Components

### Cards
Use Cards to highlight important information and link to related content:

```md
<Card
  title="Quick Start"
  icon="rocket"
  href="/quickstart"
  horizontal
>
  Get started with gx402 in minutes
</Card>
```

### Card Groups
Organize multiple cards in a grid:

```md
<CardGroup cols={2}>
<Card
  title="Unity Guide"
  icon="unity"
  href="/platforms/unity"
>
  Complete Unity integration guide
</Card>

<Card
  title="Unreal Engine Guide"
  icon="unreal"
  href="/platforms/unreal-engine"
>
  Full Unreal Engine integration
</Card>
</CardGroup>
```

### Steps
Guide users through processes with numbered steps:

```md
<Steps>
<Step title="Install the SDK">
  Follow platform-specific installation instructions
</Step>
<Step title="Initialize the SDK">
  Initialize with your API key
</Step>
<Step title="Start Using Features">
  Implement x402 functionality in your game
</Step>
</Steps>
```

### Tabs
Show examples for multiple platforms or languages:

```md
<Tabs>
<TabItem value="javascript" label="JavaScript">
```javascript
import { Gx402SDK } from '@gx402/web-sdk';

const gx402 = new Gx402SDK({
  apiKey: 'your-api-key'
});
```
</TabItem>

<TabItem value="csharp" label="C#">
```csharp
using Gx402;

Gx402Manager.Initialize("your-api-key");
```
</TabItem>

<TabItem value="cpp" label="C++">
```cpp
#include "Gx402SDK.h"

UGx402Manager::Initialize(Config);
```
</TabItem>
</Tabs>
```

### Accordion
Show detailed information that can be expanded/collapsed:

```md
<AccordionGroup>
<Accordion icon="code" title="Implementation Details">
  Detailed implementation information goes here.
  This content is hidden by default and can be expanded.
  
  ```javascript
  // Example code inside accordion
  const result = await gx402.processX402Data(data);
  ```
</Accordion>
</AccordionGroup>
```

### Columns
Display content side by side:

```md
<Columns cols={2}>
<div>

### Left Column

Content for the left column

</div>
<div>

### Right Column

Content for the right column

</div>
</Columns>
```

  apiKey: 'your-api-key',

Document API parameters:

```md
<ResponseField name="apiKey" type="string" required>

</ResponseField>

Whether to enable real-time synchronization features.

</ResponseField>

## Writing Effective Documentation

### Content Structure
Follow this pattern for most documentation pages:

```md
---
title: "Page Title"
description: "Brief description of content"
icon: "icon-name" // Optional
---

# Main Topic

Introductory paragraph explaining what this page covers.

## Main Section

Detailed information with appropriate components.

### Subsection

More specific details with code examples.

## Next Steps

Point users to related content or next actions.
```

### Code Examples Best Practices
- Use realistic, practical examples
- Include multiple platforms when relevant
- Show both success and error handling
- Keep examples concise but complete
- Use consistent variable names and patterns

```javascript
// Good: Complete example with error handling
try {
  const result = await gx402.processX402Data({
    gameData: playerData,
    options: { sync: true }
  });
  console.log('Success:', result);
} catch (error) {
  console.error('Processing failed:', error.message);
}
```

### Linking Between Pages
Use relative paths for internal links:

```md
See the [Quick Start Guide](/quickstart) for initial setup.

For platform-specific details, check the [Unity Guide](/platforms/unity).

Advanced users should read about [Security Features](/features/security).
```

## Examples

### API Documentation Page
```md
---
title: "Authentication API"
description: "API methods for user authentication"
---

# Authentication API

The authentication API provides methods for user verification.

## authenticate(userId, token)

Authenticate a user with the service.

<CodeGroup>
```javascript
const result = await gx402.authenticate('user-id', 'token');
```
```csharp
Gx402Manager.Authenticate('user-id', 'token', callback);
```
</CodeGroup>

<Note>
  Authentication must be performed before using other x402 features.
</Note>

### Parameters
- `userId` (string): Unique user identifier
- `token` (string): Authentication token

### Returns
- `Promise<Object>`: Authentication result
</Note>
```
