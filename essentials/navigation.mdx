# Navigation Structure

Learn how to organize your gx402 SDK documentation effectively using the Mintlify navigation system.

## Navigation Overview

The gx402 documentation uses Mintlify's navigation system to organize content across multiple platforms, features, and use cases.

## Basic Navigation Structure

The main navigation is configured in `docs.json` and follows this structure:

```json
{
  "navigation": {
    "tabs": [
      {
        "tab": "Getting Started",
        "groups": [
          {
            "group": "Overview",
            "pages": [
              "index",
              "quickstart",
              "what-is-gx402"
            ]
          },
          {
            "group": "Platform Guides",
            "pages": [
              "platforms/unity",
              "platforms/unreal-engine",
              "platforms/webgl",
              "platforms/telegram",
              "platforms/farcaster",
              "platforms/mobile"
            ]
          }
        ]
      }
    ]
  }
}
```

## Tab Organization

### Getting Started Tab
Organize introductory content in the "Getting Started" tab:

- **Overview**: Basic introduction and quick start
- **Platform Guides**: Platform-specific setup instructions
- **Tutorials**: Step-by-step learning guides

### SDK Reference Tab
Structure API and technical reference content:

- **API Documentation**: Complete API reference
- **Features**: Detailed feature documentation
- **Configuration**: Setup and configuration guides

### Tutorials Tab
Organize learning content by skill level:

- **Beginner**: Basic concepts and simple integrations
- **Advanced**: Complex implementations and optimizations
- **Use Cases**: Specific implementation scenarios

## Mobile Navigation Considerations

### Tab Optimization
For mobile users, consider:

- Short, descriptive tab names
- Logical grouping to minimize scrolling
- Progressive disclosure of complex options

### Content Hierarchy
```
Mobile-Optimized Navigation:
├── Quick Start (2-3 steps only)
├── Platform Guides (collapsible by platform)
├── Core Features (top 3 features visible)
├── API Reference (searchable)
└── Support (chat shortcut)
```

## Search Optimization

### Content Titles and Descriptions
Ensure each page has clear titles and descriptions for search:

```md
---
title: "Unity Integration Guide"
description: "Complete guide to integrating gx402 SDK with Unity projects"
---

# Unity Integration Guide
```

### Navigation Labels
Use descriptive but concise labels:

- ✅ "Real-time Sync" 
- ✅ "Multiplayer Features"
- ❌ "RT Sync" 
- ❌ "MP Features and Tools for Real-time Multiplayer Gaming"

## Best Practices

### Navigation Naming
- Use consistent terminology throughout
- Match user mental models ("Authentication" vs "Login")
- Keep labels scannable and clear
- Avoid jargon in navigation labels

### Grouping Content
- Group by user intent rather than technical similarity
- Limit groups to 5-7 items for cognitive load
- Use progressive disclosure for complex topics
- Maintain logical content flow between pages

### Cross-Platform Consistency
- Maintain consistent navigation structure across platforms
- Use same terminology for similar features
- Provide platform-specific context while keeping navigation similar
- Include "See also" sections for platform differences

## Global Navigation Elements

### Header Links
Add important external links in the global navigation:

```json
{
  "navigation": {
    "global": {
      "anchors": [
        {
          "anchor": "GitHub",
          "href": "https://github.com/gx402",
          "icon": "github"
        },
        {
          "anchor": "Discord",
          "href": "https://discord.gg/gx402",

    }
  }
```

## Testing Navigation

### User Flow Testing
- Test that users can reach all important content in 3 clicks or less
- Verify navigation works on mobile and desktop
- Ensure search results lead to appropriate content
- Check that "Next Steps" make logical sense

### Accessibility Considerations
- Keyboard navigation works through all navigation elements  
- Screen readers can properly interpret the navigation
- Semantic HTML structure is maintained
- ARIA labels are appropriate for complex navigation

The navigation structure should make it easy for users to find what they need, whether they're beginners getting started or experienced developers looking for specific API details.
