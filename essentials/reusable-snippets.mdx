# Reusable Snippets

Learn how to create and use reusable content snippets in your gx402 SDK documentation to maintain consistency and reduce duplication.

## Introduction to Reusable Snippets

Reusable snippets allow you to define content once and use it across multiple documentation pages. This is especially valuable for:

- Common setup instructions that are identical across platforms
- Standard error handling patterns
- API configuration examples
- Legal disclaimers and terms
- Frequently used code patterns

## Creating Snippets

### Snippet Directory Structure

Create a `snippets` directory in your docs root:

```
docs/
├── snippets/
│   ├── common/
│   │   ├── error-handling.mdx
│   │   ├── authentication-steps.mdx
│   │   └── initialization.mdx
│   ├── platforms/
│   │   ├── unity-setup.mdx
│   │   ├── unreal-setup.mdx
│   │   └── webgl-setup.mdx
│   ├── api/
│   │   ├── rate-limiting.mdx
│   │   └── response-structure.mdx
│   └── legal/
│       └── terms-of-service.mdx
```

### Basic Snippet Format

A snippet is just a regular MDX file with reusable content:

**snippets/common/initialization.mdx**
```md
## SDK Initialization

All gx402 SDK implementations require proper initialization before use:

<Tabs>
<TabItem value="javascript" label="JavaScript">
```javascript
import { Gx402SDK } from '@gx402/web-sdk';

const gx402 = new Gx402SDK({
  apiKey: 'your-api-key',
  enableRealtimeSync: true
});

await gx402.init();
```
</TabItem>

<TabItem value="csharp" label="C#">
```csharp
using Gx402;

Gx402Manager.Initialize("your-api-key");

Gx402Manager.Configure(new Gx402Config {
    EnableRealtimeSync = true
});
```
</TabItem>

<TabItem value="cpp" label="C++">
```cpp
#include "Gx402SDK.h"

FGx402Config Config;
Config.ApiKey = TEXT("your-api-key");
Config.bEnableRealtimeSync = true;

UGx402Manager::Initialize(Config);
```
</TabItem>
</Tabs>

<Note>
Remember to only initialize the SDK once in your application lifecycle.
</Note>
```

## Using Snippets in Documentation

### Import Method
For systems that support direct imports:

```md
import InitializationSnippet from '../snippets/common/initialization.mdx'

# My Documentation Page

Content before the snippet.

<InitializationSnippet />

Content after the snippet.
```

### Include Method
For systems that support file inclusion:

```md
# My Documentation Page

Content before the snippet.

{% include "./snippets/common/initialization.mdx" %}

Content after the snippet.
```

### Markdown Reference Method
For systems that support variable-like references:

```md
# My Documentation Page

Content before the snippet.

{{INITIALIZATION_CONTENT}}

Content after the snippet.
```

## Practical Snippet Examples

### Common Code Patterns

**snippets/common/error-handling.mdx**
```md
### Error Handling Best Practices

Always implement proper error handling for gx402 operations:

```javascript
const safeGx402Operation = async (operation, fallback = null) => {
  try {
    return await operation();
  } catch (error) {
    console.error('gx402 operation failed:', error);
    
    // Log to error tracking service
    logError(error);
    
    // Return fallback or rethrow based on error type
    if (error.code === 'NETWORK_ERROR') {
      return fallback;
    }
    
    throw error;
  }
};

// Usage
const result = await safeGx402Operation(
  () => gx402.processX402Data(data),
  getDefaultResult(data)
);
```

<Warning>
Never ignore errors from SDK operations as they may indicate security issues or data integrity problems.
</Warning>
```

### Required Permissions

For Unity projects, ensure the following permissions are set:

```csharp
// Add to your manifest or build settings
<uses-permission android:name="android.permission.INTERNET" />
<uses-permission android:name="android.permission.ACCESS_NETWORK_STATE" />

<meta http-equiv="Content-Security-Policy"
