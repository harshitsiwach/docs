# WebGL Integration

The gx402 SDK provides full JavaScript support for WebGL applications, enabling x402 functionality in web-based games and experiences.

## Installation

### NPM Package

```bash
npm install @gx402/web-sdk
```

### CDN

Include the SDK directly in your HTML:

```html
<script src="https://unpkg.com/@gx402/web-sdk/dist/gx402.min.js"></script>
```

### ES6 Import

```javascript
import { Gx402SDK } from '@gx402/web-sdk';
// or
import * as Gx402 from '@gx402/web-sdk';
```

## Basic Setup

### Module Import

```javascript
import { Gx402SDK } from '@gx402/web-sdk';

// Initialize the SDK
const gx402 = new Gx402SDK({
  apiKey: 'your-api-key',
  environment: 'production', // or 'development'
  enableRealtimeSync: true
});

// Wait for initialization
gx402.init()
  .then(() => {
    console.log('gx402 SDK initialized successfully');
  })
  .catch((error) => {
    console.error('Failed to initialize gx402 SDK:', error);
  });
```

### Global (CDN)

```javascript
// When using the global script, the SDK is available as
const gx402 = new window.gx402.Gx402SDK({
  apiKey: 'your-api-key'
});

gx402.init().then(() => {
  // SDK ready to use
});
```

## Core Features

### Authentication
```javascript
// Authenticate user
gx402.authenticate('user-id', 'auth-token')
  .then((result) => {
    console.log('Authentication successful:', result);
  })
  .catch((error) => {
    console.error('Authentication failed:', error);
  });
```

### x402 Functionality
```javascript
// Process data with x402 functionality
const x402Result = await gx402.processX402Data({
  gameData: yourGameData,
  options: {
    sync: true,
    encrypt: true
  }
});
```

### Real-time Synchronization
```javascript
// Subscribe to real-time updates
gx402.subscribe('game-state', (data) => {
  console.log('Received game state update:', data);
});

// Send real-time data
gx402.publish('game-state', {
  playerId: '123',
  position: { x: 100, y: 200 },
  health: 100
});
```

## WebGL-Specific Optimizations

### Memory Management
```javascript
// Optimize for WebGL memory constraints
gx402.configure({
  memoryLimit: '100MB',
  garbageCollection: true
});
```

### Performance Monitoring
```javascript
// Monitor performance
gx402.on('performance', (metrics) => {
  console.log('Performance metrics:', metrics);
});
```

## Web Workers Support

For heavy computations, the SDK supports Web Workers:

```javascript
// Initialize SDK in a Web Worker
const worker = new Worker('gx402-worker.js');

worker.postMessage({
  command: 'initialize',
  apiKey: 'your-api-key'
});
```

## Best Practices

- Minimize network requests by batching operations
- Use local caching for frequently accessed data
- Implement proper error handling for network failures
- Monitor bundle size impact
- Test across different browsers and devices
